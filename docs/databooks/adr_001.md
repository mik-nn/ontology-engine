# ADR-001: LLM as Atomic Executor Only

**Status:** Accepted

## Context

Building AI-assisted pipelines risks "LLM sprawl" — where the model makes orchestration decisions, decides next steps, manages state, and pushes to external systems. This creates unpredictable, hard-to-audit behavior.

## Decision

LLMs in this engine are **atomic task executors only**:
- They receive a fully-formed, SHACL-validated context packet
- They return a structured, schema-bound result
- They never see the graph directly, never decide next steps, never call tools autonomously
- All orchestration is done by deterministic Python agents reading ontology state

## Consequences

- Every LLM call is auditable: input context and output result are both logged as RDF events
- Pipeline is reproducible: same graph state + same rules → same plan
- LLM can be swapped (GPT-4 → Claude → local model) without changing orchestration logic
- Debugging is easier: when something goes wrong, the cause is in the graph or rules, not an LLM chain
